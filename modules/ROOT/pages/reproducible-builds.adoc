= 再現可能なビルド
ifndef::env-site,env-github[]
include::_attributes.adoc[]
endif::[]

特定のバージョンの Mule プロジェクトはいつでもビルドできます。これにより、本番環境のプロジェクトが常に最新とは限らないため、メンテナンスが容易になります。

この目標を達成するには、次の手順を実行します。

* すべての Mule プロジェクトのソース制御を使用し、分岐、マージ、タギングをプロジェクトに適用します。(Eclipse 固有のファイルではチェックしないでください)。
* Maven と、Nexus などの社内リポジトリマネージャを使用して連動関係を厳密に管理します。
