= Mule アプリケーションのデプロイメントモデル

include::_attributes.adoc[]

:keywords: Anypoint Studio, esb, デプロイ, アーキテクチャ

デプロイメント用 Mule プロジェクトは Mule アプリケーションです。Mule Runtime インスタンスは、同じ Mule コンテナ内で独立して実行される複数のアプリケーションをサポートしています。これにより、次のような追加のメリットが得られます。

* 複雑なアプリケーションをビルドする必要がある場合に、特定のロジックが含まれる複数のアプリケーションに分割してから、アプリケーションを同じ Mule コンテナ内でデプロイすることができます。
* Mule アプリケーションの実体で操作を実行するための境界をクリアできます。
* Mule はアプリケーションを監視でき、設定変更を自動的に再読み込みします。
* 以前に競合したことがあったとしても、アプリケーションは異なるライブラリバージョンに依存できます。
* 複数のバージョンのアプリケーションを同じ Mule コンテナ内で実行できます。

すべてのアプリケーションは​*ドメイン*​内で一緒にデプロイされます。デフォルトでは、アプリケーションはデフォルトのドメインを参照し、追加の設定は必要ありません。コンソールでアプリケーションと一緒にデプロイされるのを確認できます。

image::deploy-domain.png[deploy+domain]

複数のアプリケーションを同じ場所にデプロイしていて、アプリケーションが同じリソースを共有できる場合は、共通の​*ドメイン*​を作成して複数のプロジェクトから参照できる設定を定義できます。これにより、競合を起こすことなく、同じ HTTP ホストおよびポートから異なるプロジェクトの異なるサービスを公開できます。
