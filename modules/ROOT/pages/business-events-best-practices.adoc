= ビジネスイベントのベストプラクティス

:keywords: Anypoint, Studio, esb, ビジネスイベント

Mule アプリケーションでビジネスイベント追跡を設定する場合、いくつかの推奨方法があります。

* 特定の値のあるプロセスの場合にのみデフォルトイベントを有効にする。デプロイする前にビジネストランザクション内のどのフェーズを追跡するのかを決定し、それらのフェーズの追跡を有効にします。すべてのイベント候補を追跡することもできますが、本当に分析する必要があるイベントを実行時に見つけるためのイベントの絞り込みやクエリにより多くの時間がかかります。
* カスタムイベントを使用して、プロセスの重要指標 (注文合計金額や追跡番号など) を追跡し、フローの高レベルのビジネスアクティビティを明らかにする。
* トランザクションの有用な情報 (注文番号、従業員 ID 番号、出荷追跡番号など) が表示されるように、トランザクション ID をカスタマイズする。これにより、Mule 管理コンソールと CloudHub のどちらを使用している場合でも、実行時の分析やデバッグを簡単かつ直感的に行うことができるようになります。
